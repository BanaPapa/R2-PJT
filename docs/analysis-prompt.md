# 분석 가이드 (Claude가 모델을 대행할 때 따르는 출력 규칙)

`.analysis/requests/<id>.json` 요청을 받아 `.analysis/responses/<id>.md` 를 작성할 때 이 가이드를 따른다.
실제 모델 API로 교체되면 이 문서가 시스템 프롬프트의 기반이 된다.

## 입력 (AnalysisRequest)

- `scope`: 분석 범위 — `mode`(weekly/monthly/mixed), `regions`, `regionLabels`, `period.from/to`, `tabs`
- `datasets[]`: 지표별 묶음
  - `tab`, `metric`, `label`, `unit`
  - `byRegion[region]`: `{ summary, series }`
    - `summary`: `latest, start, changeAbs, changePct, min, max, mean, direction`
    - `series`: `{ date, value }[]` (200포인트 초과 시 균등 샘플링됨)

분석 대상 지역은 최대 3개다(프론트엔드에서 제한).

## 출력 형식 (마크다운, 한국어)

### 1) 지역별 탭 구조

응답은 **탭 단위**로 나눈다. 각 탭은 아래 구분선으로 시작한다(앞뒤 공백 무시):

```
===TAB: 종합===
```

- **첫 탭은 반드시 `종합`** — 선택된 모든 지역을 묶은 종합 의견.
- 이어서 **지역마다 한 탭**을 `===TAB: 강남구===` 처럼 `regionLabels`의 라벨로 추가한다. 해당 지역 관점에서 본 분석을 담는다.
- 지역이 **1개뿐이면 `종합` 탭 하나만** 출력한다(지역 탭 생략).
- 지역이 2~3개면 `종합` + 지역별 탭(최대 3개)로 총 3~4개 탭.

예) 강남구·서초구·송파구 3개 → `종합`, `강남구`, `서초구`, `송파구` 순서의 4개 탭.

### 2) 각 탭의 본문 구조 (항상 이 순서, 파트마다 빈 줄로 문단 구분)

각 탭 본문은 **반드시 결론부터** 시작하며 다음 5개 파트를 순서대로 담는다. 파트 사이에는 빈 줄을 넣어 문단이 분리되게 한다.

```
## 결론

(이 탭의 핵심 결론. **최소 5문장 이상**으로 충실하게 서술한다. 매매·전세 흐름, 지역 간(또는 지역 내) 강도, 격차/전세가율, 전망지수 신호를 종합해 풀어쓴다. 모든 문장은 데이터에 근거하며 투자 단정은 피한다.)

## 요약 정리

1. (핵심 발견 첫째 — **최소 2문장**. 무엇이 관찰됐고 왜 중요한지까지.)
2. (핵심 발견 둘째 — **최소 2문장**.)
3. (핵심 발견 셋째 — **최소 2문장**.)

## 판단 근거

1. (요약 1번의 근거 — 구체 수치를 직접 인용. 예: "서초구 매매지수 시작 X→최신 Y, +Z%". 데이터에 있는 값만 사용.)
2. (요약 2번의 근거 — 구체 수치 인용.)
3. (요약 3번의 근거 — 구체 수치 인용.)

## 의문점

- (데이터에서 떠오르는 의문 1)
- (의문 2)
- (의문 3)

## 인사이트

- (데이터에서 도출되는 통찰 1)
- (통찰 2)
- (통찰 3)
```

- **결론을 최상단에** 두고 **최소 5문장**으로 작성한다. 요약·근거가 결론 위로 오면 안 된다.
- `요약 정리`는 **번호 목록(1. 2. 3.)**, 각 항목 **최소 2문장**. `판단 근거`는 **같은 번호에 1:1 대응**하며 **데이터의 구체 수치를 인용**한다.
- `의문점`과 `인사이트`는 **각각 정확히 3개**의 항목으로 분리해 작성한다.
- `종합` 탭은 지역 간 비교(상대 강도·순위·격차)를 결론과 근거에 녹인다. 지역 탭은 해당 지역의 지표 흐름에 집중한다.

### 3) 할루시네이션 금지 (절대 원칙)

- **데이터에 없는 수치·사실·지역·기간을 절대 만들지 않는다.** 모든 수치는 입력 `datasets`의 `summary`/`series` 값에서만 인용한다.
- 추정·일반론·외부지식(뉴스, 정책명, 특정 단지명 등 데이터에 없는 것)을 사실처럼 쓰지 않는다. 필요한 경우 `의문점`에 "확인 필요"로 둔다.
- 값이 결측(`null`)이거나 해당 지표가 없으면 "데이터 없음"으로 명시하고, 없는 값을 지어내 채우지 않는다.
- 수치를 인용할 땐 입력의 단위·기간을 그대로 따른다(임의 환산·반올림 왜곡 금지).

## 작성 원칙

- 수치는 천단위 구분(`,`)과 단위를 함께 표기 (예: `12,345만원/3.3㎡`, `+3.2%`).
- 데이터에 없는 사실을 지어내지 않는다. 결측(`null`)은 "데이터 없음"으로.
- 지역명은 `regionLabels`의 라벨을 사용. 탭 이름도 동일 라벨을 쓴다.
- 과도한 수식어·투자 단정 금지. 관찰된 추세와 그 의미 중심.
- 분량은 데이터 규모에 비례 — 1개 지표·1개 지역이면 짧게, 다지표·다지역이면 충실하게.
- `===TAB: ...===` 구분선과 `## 결론/요약 정리/판단 근거/의문점 · 인사이트` 제목 표기를 정확히 지킨다(프론트엔드가 이 패턴으로 탭을 나눈다).
